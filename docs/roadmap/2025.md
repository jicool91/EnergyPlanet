|--------|-------|-----------------|
| Q1 2025 | Сезонная система v1 | Бэкенд + UI готовы. Боевой пропуск включён (`feat: launch battle pass progression`), автоскрипт выдачи наград доступен (`feat: automate season reward payouts`). |
| Q1 2025 | Телеметрия UI | Уже внедрены события `render_latency`, `safe_area_changed`, `viewport_action`. **Статус:** ждёт OpenTelemetry + Grafana отчётов. |

| Q2 2025 | PvP / Arena | Контент `content/events/pvp.json` и эндпоинты `/events/pvp*` запущены (`feat: expose PvP events API`). Следующий шаг — матчмейкер и симуляция боёв на схеме `arena_*`. |
| Q3 2025 | Battle Pass | Вертикальный срез: покупка премиума, claim tier rewards, SeasonScreen. Осталось расширить экономику/платежи. |

| Q4 2025 | Social 2.0 | API друзей `/friends` реализован (`feat: implement friends API`), нужно real-time и дополнительные UX. |
| Q4 2025 | Infra hardening | HA PostgreSQL, Redis cluster, Canary deploys, auto-scaling фронта, логирование в Loki — TODO. |

## Неприоритезированные идеи
- Push-уведомления (если Telegram разрешит).
- PvE «арены» (юзать `arena_events`).
- LiveOps-инструменты для массовой выдачи наград (расширить `AdminService`).
- A/B эксперименты (уже есть `experiments` в `flags`, нужно UI и сбор статистики).
