# Roadmap 2025

| Период | Фокус | Детали |
|--------|-------|--------|
| Q1 2025 | Сезонная система v1 | Завершили `season_progress`/`season_rewards`/`season_events` (миграция 017). Нужно: UI боевого пропуска, cron для выдачи наград, админ-интерфейс. |
| Q1 2025 | Телеметрия UI | Уже внедрены события `render_latency`, `safe_area_changed`, `viewport_action`. План: интегрировать с OpenTelemetry, строить отчёты на Grafana. |
| Q2 2025 | Кланы (соц-геймплей) | Таблицы `clans`, `clan_members`, `clan_chat`, `clan_events`, `clan_perks` (миграция 002) готовы. Задачи: API/сервисы, UI `ClanScreen`, модерация. |
| Q2 2025 | PvP / Arena | Таблицы `arena_*` (миграция 003). Нужно реализовать матчмейкер, симуляцию боёв, UI (`PvPEventsScreen`). |
| Q3 2025 | Battle Pass | Основание в `season` YAML (`battle_pass`). Задачи: контентная модель, клиентский UI, серверные награды, платежи. |
| Q3 2025 | Monetization v2 | Улучшить `PurchaseService` (реальная Telegram Stars интеграция, webhooks), автоматические отчёты (`MonetizationAnalyticsService`). |
| Q4 2025 | Social 2.0 | Друзья/рефералы/лидерборды — пересмотреть ограничения, добавить события в real-time (push). |
| Q4 2025 | Infra hardening | HA PostgreSQL, Redis cluster, Canary deploys, auto-scaling frontend, логирование в Loki, расширение alerting.

## Неприоритезированные идеи
- Push-уведомления (если Telegram разрешит).
- PvE «арены» (юзать `arena_events`).
- LiveOps-инструменты для массовой выдачи наград (расширить `AdminService`).
- A/B эксперименты (уже есть `experiments` в `flags`, нужно UI и сбор статистики).

Обновляйте таблицу при изменении приоритетов. Каждая крупная фича должна иметь ссылку на миграции/код и документацию.
