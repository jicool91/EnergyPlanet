# DS UX Audit Survey (Stage E follow-up) — 2025-11-07

## Назначение
После миграции экранов на дизайн-систему Stage E/F собираем обратную связь от продуктовых и UI-команд. Анкета закрывает требование чеклиста Stage E п.4 и фиксирует метрики удовлетворённости.

## Куда размещена форма
- **Форма:** https://forms.gle/energy-planet-ds-ux (скопируйте в Notion при необходимости офлайн-доступа).
- **Ответственные:** UI Team A (owner), DS Tech Lead (backup). Проверка раз в пятницу.

## Структура анкеты
1. **Профиль участника**
   - Команда / роль (дропдаун: UI, Backend, QA, PM, Support).
   - Какие экраны вы трогали после миграции? (мультивыбор).
2. **Оценка восприятия (Likert 1‑5)**
   - Насколько легко стало дорабатывать экран?
   - Удовлетворены ли вы визуальной консистентностью?
   - Стало ли проще писать тесты (Playwright/Chromatic)?
3. **Доступность**
   - Были ли проблемы с фокусом/контрастом? (да/нет + описание).
   - Проверяли ли вы `prefers-reduced-motion`? (да/нет).
4. **Техническая часть**
   - Каких токенов/компонентов не хватало?
   - Сколько времени занял перенос? (диапазон).
5. **Свободные комментарии / ссылки на баги**

## Интеграция
- Результаты уходят в Notion Database `Design System Feedback` (см. `notion://ds-feedback`).
- Скрипт `scripts/export-ux-audit.js` преобразует Google Sheets → JSON для аналитики.
- Еженедельный дайджест публикуется в `#ds-rollout` (Slack) по шаблону: TL;DR, blockers, апрувы.
